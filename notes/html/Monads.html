Monads and Me


When you see a function with a signature like


appraisalUI :: MonadIO m => AppraisalUI -> MyView -> Alderon MyView m ()


the portion of the return type which shown here in red is a type with one parameter, that is, a type whose kind is * -> *. My mental model for understanding what the red portion of the type signature means is pretty clear in cases like Set or Maybe, but it is much less clear for complicated types like the one above. Particularly magical for me are monads like StateT, which make it seem like that red portion is a container with a persistent value which you can access with get and put.


This is a fine way of thinking about it when you are using the monad, but it seems to confuse me when I am trying to modify it or write a MonadState instance.
